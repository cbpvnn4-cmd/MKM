الطريقة 1: الملف العادي ✅ (موصى به)
────────────────────────────────────────
📄 start_backend.bat

• اضغط عليه مرتين
• يفحص إذا كان السيرفر يعمل
• يعرض الأخطاء إن وجدت
• النافذة تبقى مفتوحة لترى الأخطاء


الطريقة 2: الملف المفصّل 🔍 (للتشخيص)
────────────────────────────────────────
📄 start_backend_verbose.bat

• يعرض كل خطوة بالتفصيل
• يفحص Python
• يفحص الملفات
• يعرض الأخطاء بألوان واضحة
• يعطي حلول للمشاكل الشائعة

👉 استخدم هذا الملف عندما تواجه مشاكل!


الطريقة 3: الملف القديم المحسّن
────────────────────────────────────────
📄 start_backend_only.bat

• نفس start_backend.bat لكن مُحسّن
• يعمل بنفس الطريقة

المشكلة 1: "SERVER ALREADY RUNNING"
────────────────────────────────────────
✗ السيرفر يعمل بالفعل!

الحل:
1. أغلق نافذة السيرفر القديمة
2. أو شغل: stop_backend.bat
3. ثم جرب مرة أخرى


المشكلة 2: "Port 8000 already in use"
────────────────────────────────────────
✗ البورت 8000 مشغول!

الحل:
1. شغل stop_backend.bat
2. أو افتح CMD وأكتب:

   netstat -ano | findstr :8000

   ثم:

   taskkill /F /PID <رقم_البروسس>


المشكلة 3: "Python not found"
────────────────────────────────────────
✗ Python غير مثبت أو غير موجود في PATH!

الحل:
1. تأكد من تثبيت Python
2. أعد تشغيل CMD
3. جرب: python --version


المشكلة 4: "Module not found" أو "ImportError"
────────────────────────────────────────
✗ مكتبات Python مفقودة!

الحل:
1. افتح CMD في مجلد backend
2. شغل:

   pip install -r requirements.txt


المشكلة 5: "backend directory not found"
────────────────────────────────────────
✗ تشغل الملف من مكان خاطئ!

الحل:
• تأكد أن الملف .bat موجود في المجلد الرئيسي
• يجب أن يكون بجانب مجلد backend


المشكلة 6: النافذة تقفل بسرعة
────────────────────────────────────────
✗ لا تشوف الأخطاء!

الحل:
• استخدم start_backend_verbose.bat
• أو شغل من CMD مباشرة

🔍 كيف تعرف السيرفر يشتغل صح؟
═══════════════════════════════════════════════════════

✅ العلامات الصحيحة:

1. النافذة تبقى مفتوحة
2. ترى رسالة:
   "Uvicorn running on http://0.0.0.0:8000"
3. لا توجد رسائل خطأ حمراء
4. تقدر تفتح المتصفح:
   http://localhost:8000/docs

═══════════════════════════════════════════════════════
📝 ملاحظات مهمة
═══════════════════════════════════════════════════════

• لا تقفل نافذة السيرفر وهو يشتغل!

• إذا غيّرت الكود، لازم تعيد تشغيل السيرفر
  (أو استخدم reload=True في الكود)

• السيرفر يشتغل على:
  - http://localhost:8000
  - http://127.0.0.1:8000
  - http://0.0.0.0:8000 (داخلياً)

• API Docs موجودة على:
  http://localhost:8000/docs

🎬 مثال: الاستخدام الصحيح
═══════════════════════════════════════════════════════

خطوة 1: تشغيل السيرفر
──────────────────────
• اضغط مرتين على: start_backend.bat
• انتظر لحد ما تشوف:
  "Uvicorn running on..."

خطوة 2: اختبار
──────────────────────
• افتح المتصفح
• روح على: http://localhost:8000/docs
• لازم تشوف صفحة API Documentation

خطوة 3: شغل Frontend
──────────────────────
• افتح نافذة CMD ثانية
• cd frontend
• npm run dev

خطوة 4: اختبار النظام
──────────────────────
• افتح المتصفح على: http://localhost:5173
• سجل دخول
• جرب الفواتير

خطوة 5: الإيقاف
──────────────────────
• اضغط Ctrl+C في نافذة السيرفر
• أو شغل: stop_backend.bat

═══════════════════════════════════════════════════════
💡 نصائح احترافية
═══════════════════════════════════════════════════════

1. استخدم start_backend_verbose.bat أول مرة
   (عشان تتأكد كل شي يشتغل صح)

2. بعدين استخدم start_backend.bat للاستخدام اليومي

3. اترك نافذة السيرفر مفتوحة دائماً
   (عشان تشوف أي أخطاء تصير)

4. لو صار أي خطأ، شوف آخر سطر في النافذة
   (عادة يقول لك وين المشكلة)

5. لو مو فاهم الخطأ، انسخه وابحث عنه
   (أو ارسله لـ Claude!)
