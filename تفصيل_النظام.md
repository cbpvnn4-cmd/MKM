# 📚 توثيق نظام السند للمصاعد الشامل

## SANAD ELEVATORS - Comprehensive Management System

---

## 📌 نظرة عامة على النظام

**نظام السند للمصاعد** هو نظام متكامل لإدارة شركات المصاعد يشمل:

- إدارة العمليات التجارية (مبيعات، مشتريات، عقود)
- إدارة الشركاء وتوزيع الأرباح
- إدارة المخزون والمستودعات
- إدارة العملاء والموردين
- نظام التقارير المالية الشاملة
- إدارة الصيانة والدعم الفني

### 🛠️ التقنيات المستخدمة

| الطبقة                | التقنية                                   |
| --------------------- | ----------------------------------------- |
| **Backend**           | Python 3.x + FastAPI                      |
| **Frontend**          | React.js + Vite + TailwindCSS             |
| **Database**          | PostgreSQL 15 (Docker) / SQLite (للتطوير) |
| **Authentication**    | JWT (JSON Web Tokens)                     |
| **Containerization**  | Docker + Docker Compose                   |
| **API Documentation** | Swagger/OpenAPI (FastAPI auto-generated)  |

## 🗄️ هيكل قاعدة البيانات (Data Models)

### 1️⃣ نموذج المستخدمين والصلاحيات (Users & RBAC)

```
📊 users
├── id (PK)
├── username (unique)
├── email (unique)
├── hashed_password
├── full_name
├── phone
├── is_active
├── is_superuser
├── last_login
└── created_at, updated_at

📊 roles
├── id (PK)
├── name (unique) - Admin, Manager, Accountant, Sales, User
├── description
└── is_active

📊 user_roles (many-to-many)
├── user_id (FK → users)
└── role_id (FK → roles)

📊 permissions
├── id (PK)
├── name
├── description
├── resource (users, partners, sales, etc.)
└── action (create, read, update, delete)

📊 role_permissions
├── role_id (FK → roles)
└── permission_id (FK → permissions)
```

### 2️⃣ نموذج الشركاء (Partners)

```
📊 partners

└── is_active

📊 partner_capital_movements

├── movement_type (DEPOSIT, WITHDRAW)
├── amount_usd
├── happened_at
└── note

📊 ownership_snapshots

├── equity_usd
└── equity_pct (0-100%)

📊 partner_account_ledger
├── partner_id (FK)
├── transaction_date
├── transaction_type (CAPITAL_DEPOSIT, CAPITAL_WITHDRAW, PROFIT_DISTRIBUTION, PROFIT_PAYOUT, ADJUSTMENT)
├── debit_amount, credit_amount

└── description

📊 partner_profit_payouts
├── partner_id (FK)
├── distribution_line_id (FK)
├── payout_amount
├── payment_method (BANK_TRANSFER, CASH, CHECK, WIRE_TRANSFER)

```

## 🔄 آلية عمل النظام

### 1️⃣ دورة المبيعات

```
عميل → عرض سعر → عقد (اختياري) → أمر بيع → فاتورة → تحصيل
```

### 2️⃣ دورة المشتريات

```
مورد → أمر شراء → استلام بضاعة → تحديث مخزون → فاتورة مورد → دفع
```

### 3️⃣ دورة توزيع الأرباح

```
حساب صافي الربح الشهري
↓
خصم نسبة الشركة (30%)
↓
توزيع على مستفيدي الشركة
↓
توزيع الباقي (70%) على الشركاء
↓
(بناءً على رأس المال الموزون بالزمن - TWCap)
↓
تسجيل القرارات (سحب نقدي / تحويل لرأس مال)
```

### 4️⃣ حساب رأس المال الموزون بالزمن (TWCap)

```python
# لكل شريك i في الشهر:
TWCap_i = Σ (Capital_at_day × days_held) / total_days_in_month

# النسبة من إجمالي رأس المال:
share_i = TWCap_i / Σ(TWCap_all)

# حصة الشريك من الأرباح:
profit_i = partners_pool × share_i
```
